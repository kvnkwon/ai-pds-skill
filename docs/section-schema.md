# Section Schema System

Each section is a structured document.

## Structure

Each section has:

- metadata
- structured fields
- markdown output
- version history
- clarity score
- confidence score

## Core Sections

- Overview
- Problem Statement
- Target User
- JTBD
- Pain Points
- Solution Direction
- Competitor Analysis
- Market Opportunity
- MVP Scope
- Critical Path
- Assumptions
- Risks
- Technical Direction
- Claude Code Starter
- Notes

## Key Principle

Docs should feel like markdown
System should behave like structured data

## Storage

Each section stores:
- structured JSON
- generated markdown

## Updates

Chat updates fields, not full documents
