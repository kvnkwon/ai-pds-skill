# Problem and Opportunity

## Market Shift

We are in a new era where:
- building is easy (AI, no-code, vibe coding)
- thinking is the bottleneck

Users can:
- build apps quickly
- generate UI instantly
- scaffold code in minutes

But they cannot:
- define good problems
- identify the right users
- scope MVPs correctly
- avoid overbuilding
- understand tradeoffs

## Current Workflow

1. Idea
2. Open ChatGPT / Claude
3. Ask vague prompt
4. Get generic output
5. Jump into Cursor / Replit
6. Start building
7. Realize something is off
8. Restart or abandon

## Core Problems

- Prompt paralysis
- No structure
- No persistence
- Fragmented thinking
- No progression system
- No synthesis
- No transition to build
- No strategic validation

## Design Sprint Problems (from research)

Traditional design sprints fail because:
- too time compressed
- require full team availability
- require ideal conditions
- heavy coordination overhead
- often become UX theater
- lack follow-through
- produce shallow outputs
- rigid structure
- false confidence

## Opportunity

Create a system that:
- preserves benefits of design sprints
- removes dependency on teams
- works asynchronously
- maintains persistent artifacts
- adapts dynamically
- guides users step-by-step
- produces build-ready outputs

## Core Opportunity Statement

There is no structured AI system that helps users think like a product team before building.

This product becomes:
The missing layer between idea and execution.
