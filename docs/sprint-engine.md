# Sprint Engine

The product follows a progressive, stage-based system.

## Key Principle

Sections are earned, not pre-loaded.

## Stages

1. Exploration
2. Problem Definition
3. User & JTBD
4. Solution Direction
5. Market & Competitors
6. MVP & Critical Path
7. Assumptions & Risk
8. Build Handoff

## Each Stage Has

- required clarity signals
- validation logic
- unlock conditions

## Progression Logic

AI evaluates:
- clarity
- completeness
- confidence

Then:
- asks questions
- proposes artifacts
- suggests moving forward

## Example

If problem not defined:
→ ask targeted question

If defined:
→ generate Problem Statement
→ ask for confirmation

## UX Behavior

- soft gating
- never fully blocked
- user can skip
- system warns if skipping

## Goal

Guide user through correct thinking sequence.
