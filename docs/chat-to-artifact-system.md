# Chat to Artifact System

## Core Principle

Chat = thinking
Sections = decisions

## Flow

1. User chats
2. AI detects structured insight
3. AI proposes artifact
4. User confirms
5. Section updates

## Example

User:
"This is more for beginners"

System:
"Update Target User?"

## Rules

- do not auto-save blindly
- always propose before saving
- allow edit before save
- allow rejection

## Behavior

Sections are living documents
Can be updated over time

## UX Element

Inline suggestion card:

Candidate: Problem Statement
[summary]

Actions:
- Save
- Edit
- Not now
