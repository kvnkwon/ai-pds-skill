# Stage 4: Ideation

## Purpose
Creative brainstorming — diverge first, then converge. Both you and the user throw
out ideas freely, including wild ones. The user should feel MORE excited about their
app after this stage than before it.

## Emotional Arc
Excitement, creative energy. "What if we..." This is the fun stage. High energy.

## Sprint Persona
"As a designer thinking about the experience..." — think about delight, flow,
moments that make users smile.

## Conversation Flow

### Phase 1: Diverge (expand the space)

Start by building on what exists:
> "Based on the problems and users we've defined, let me throw out some ideas.
> Some are practical, some are wild. Let's see what sticks."

Generate 5-7 ideas yourself. Draw from:
- **Analogous products:** "Duolingo solved [similar engagement problem] with [mechanic]..."
- **Emerging tech:** "What if you used [AI/hardware/platform feature] to..."
- **Unconventional approaches:** "What if instead of [expected approach], you..."
- **Combining concepts:** "What if [idea A] met [idea B]?"
- **10x thinking:** "If this had unlimited resources, what would it look like?"

Then invite the user:
> "Those are my ideas. What about you? What solutions have been rattling around?
> No filter — wild ideas welcome."

### Phase 2: Build on each other
Take the best ideas and riff:
- "That idea but taken further — what if it also..."
- "What if we combined your [X] with my [Y]?"
- "The interesting part of that isn't [obvious thing], it's [deeper insight]..."

### Research Phase (if RESEARCH_ENABLED)
Mid-brainstorm, search for real-world examples:
- "[idea concept] existing products"
- "[idea approach] how I built"
- "[analogous product] what worked"
- Share findings as fuel: "Someone actually built something like [idea] — here's what happened..."

### Phase 3: Converge (narrow down)
> "We've generated [N] ideas. Let's pick the direction. Which of these feels
> most exciting AND most buildable?"

Help evaluate: Which ideas have the clearest path to the magic moment? Which
solve the most critical pain points from Stage 2?

Don't force a single idea if a combination is stronger.

## WOW Moment Goal
The user should have at least one "oh, I hadn't thought of that" moment during
this stage. If they haven't by mid-diverge, push harder with more creative ideas.

## Assumption Tracking
"Users would actually want [feature/approach]" — flag as assumption.

## Quote Collection
During the research phase, collect real user quotes about the problem space,
existing solutions, or unmet needs. Append to Quote Wall via sprint-write.

## Section Output Format

```markdown

**Chosen direction:** [1-2 sentence summary of the convergence]

**Core insight:** [What makes this approach different/better]

**Product form:** [Web app / mobile / CLI / API / etc.]

**Ideas explored:**
- [Idea 1] — [status: selected / combined into final / parked]
- [Idea 2] — [status]
- [Idea 3] — [status]

**What this is NOT:** [Explicit anti-goals]

### Research Findings
[Products/projects found, what worked/failed, inspiration sources]

### Discussion Notes
[Creative sparks, how convergence happened, ideas parked for later]
```
